# 日期格式参考

当从列表容器中提取到日期文本后，使用本文档中的格式进行解析。

> 由 `skill.md` 第六步引用。不需要单独加载——仅在 `skill.md` 执行到日期解析阶段时按需查阅。

---

## 1. 快速索引（先从这里开始）

大多数（90%+）站点只用到以下 4 种格式：

| 优先级 | 格式 | 示例 | 正则 |
|--------|------|------|------|
| 1 | ISO 标准 | `2026-05-11` | `\d{4}-\d{2}-\d{2}` |
| 2 | 斜杠分隔 | `2026/05/11`, `05/11/2026` | `\d{2,4}[/-]\d{2}[/-]\d{2,4}` |
| 3 | 中文日期 | `2026年5月11日` | `(\d{4})年(\d{1,2})月(\d{1,2})日` |
| 4 | 相对日期 | `今天`, `昨天`, `X小时前` | 见下方相对日期表 |

如果以上 4 种都不匹配，再查阅下面的完整表。

---

## 2. 数字日期格式（完整）

| 格式 | 示例 | 正则 |
|------|------|------|
| ISO 标准 | `2026-05-11` | `\d{4}-\d{2}-\d{2}` |
| 斜杠分隔 | `2026/05/11`, `05/11/2026`, `11/05/2026` | `\d{2,4}[/-]\d{2}[/-]\d{2,4}` |
| 点分隔 | `2026.05.11`, `11.05.2026` | `\d{2,4}\.\d{2}\.\d{2,4}` |
| 紧凑格式 | `20260511` | `\d{8}` |
| 中文日期 | `2026年5月11日` | `(\d{4})年(\d{1,2})月(\d{1,2})日` |
| 日文日期 | `2026年5月11日` | 同中文 |
| 韩文日期 | `2026년 5월 11일` | `(\d{4})년\s*(\d{1,2})월\s*(\d{1,2})일` |
| 英文月名 | `May 11, 2026`, `11 May 2026` | 见英文月份解析 |
| 带时间戳 | `2026-05-11 08:30`, `May 11, 2026 8:30 AM` | 先提取日期部分再解析 |

## 3. 相对日期格式（需换算为绝对日期）

| 格式 | 含义 | 换算 |
|------|------|------|
| `X 小时前` / `X hours ago` | X 小时前 | 今天 |
| `昨天` / `Yesterday` | 昨天 | T-1 |
| `前天` / `2 days ago` | 前天 | T-2 |
| `X 天前` / `X days ago` | X 天前 | T-X |
| `今天` / `Today` | 今天 | T |
| `刚刚` / `Just now` | 刚刚 | 今天 |
| `1 周前` / `1 week ago` | 一周前 | T-7 |

## 5. 多语言关键词（用于搜索定位日期元素）

### 日期关键词（按语言）

- **英文**: `date`, `time`, `publish`, `post`, `create`, `update`, `timestamp`, `when`, `posted`
- **中文**: `时间`, `日期`, `发布`, `更新`
- **日文**: `日付`, `投稿日`, `更新日`, `公開`
- **韩文**: `날짜`, `시간`, `등록일`, `작성일`

### 相对日期/星期词（用于确认而非解析）

- **中文**: 星期一~日、周一~日、今天、昨天、前天、刚刚
- **英文**: Monday~Sunday、Today、Yesterday
- **日文**: 月曜日~日曜日、今日、昨日、一昨日
- **韩文**: 월요일~일요일、오늘、어제、그제

### XPath 搜索示例

```
.//span[contains(@class, 'date')]/text()
.//div[contains(@class, 'publish-time')]/text()
.//span[@class='post-meta-time']/text()
```

## 6. 日期提取搜索优先级

在 `list_item` 容器内按以下顺序搜索：

| 优先级 | 目标 | 示例 |
|--------|------|------|
| 1 | `<time>` 标签（HTML5 语义） | `.//time/@datetime` → `"2026-05-11T08:30:00+08:00"` |
| 2 | 含日期关键词的 class/属性 | `.//span[contains(@class, 'date')]/text()` |
| 3 | 固定位置的日期容器 | `.//div[@class='item-meta']/span[1]/text()` |
| 4 | 链接 URL 中的日期 | regex: `/(\d{4})/(\d{2})/(\d{2})/` |

---

## 7. 解析流程

1. 在容器内执行日期 XPath，获取日期文本
2. 清理文本：去除前后空白、多余空格、`&nbsp;`
3. 尝试 §2~§3 所有格式的正则匹配，提取年/月/日
4. 对于相对日期，以当前日期（T）为基准换算
5. 记录每条列表项解析后的日期

**解析示例**：

```
"2026年5月11日"        → (2026, 5, 11)
"05/11/2026"           → (2026, 5, 11)  -- 注意美式月/日/年
"May 11, 2026"         → (2026, 5, 11)
"3小时前" (今天)       → T
"昨天"                 → T-1
"2026-05-11T08:30:00Z" → (2026, 5, 11)
"/2026/05/11/title"    → (2026, 5, 11)  -- URL 路径中提取
```

## 8. 日期提取失败时的渐进式兜底

按以下步骤逐级尝试：

**步骤 A**：扩大搜索范围到容器父级（有些站点日期在紧邻的兄弟元素中）

**步骤 B**：检查日期是否在列表以外的独立区域（按日期分组的站点，如 `<h3>2026年5月11日</h3>` + 若干该日期条目）

**步骤 C**：检查 JSON-LD 或 meta 标签
```
//script[@type='application/ld+json']   →  JSON 中可能含 datePublished
//meta[@property='article:published_time']/@content
```

**步骤 D**：完全无法提取日期 → 设 `date_unavailable = true`，交给分页判定逻辑的降级模式处理。
