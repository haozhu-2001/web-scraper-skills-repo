# 分页判定逻辑

本文档定义"列表抓取翻多少页"的判定方法。

> 由 `skill.md` 第六步引用。IN: 首页日期列表 + next_page 信号。OUT: `stop` 值 + 模板选择（3.3/3.4）。

---

## 核心规则

**以 [T-2, T-1] 窗口（前天到昨天）内列表项所经过的分页数，作为 `stop` 的依据。**

- T = 当天
- 时间窗口 = [T-2, T-1]

---

## 第一步：判断是否需要分页（四态判定）

### 前置信号

- `next_page`: `null` | `XPath字符串` | `"scroll"`
- 首页条目数：`list_item` XPath 命中数
- 单页容量 ≈ 首页条目数
- `date_unavailable`: 日期是否可提取（来自 `references/date-formats.md` 第 8 节步骤 D）

---

### 分支 0：日期不可提取（`date_unavailable = true`）

```
if next_page ≠ null 且 首页条目数 ≈ 单页容量:
    → 进入降级模式（见 §降级模式）
else:
    → 不分页，模板 3.3
```

---

### 分支 1：日期可提取（四态判定）

#### 状态速查

| 状态 | 首页日期特征 | 判定 |
|------|------------|------|
| **A** | 首页最旧条目 ≤ T-2 | 不分页，模板 3.3 |
| **B** | 首页有部分 [T-2,T-1]（如 T-1 在首页，T-2 在后页） | 翻页查找，模板 3.4 |
| **C-Ⅰ** | 首页无 [T-2,T-1]，next_page=null 或 首页稀疏 | 不分页，模板 3.3 |
| **C-Ⅱ** | 首页无 [T-2,T-1]，next_page≠null 且 首页塞满 | 升级为 B，翻页查找 |

#### 状态 C 为什么需要裂开？

低频站点（几天一篇）和高频站点（每天几十篇）的首页表现相同——都只有当天内容。区别在于：

- **C-Ⅰ（低频）**: 真的只有这点内容，翻页没用
- **C-Ⅱ（高频）**: 内容被当天的条目挤出，翻页才能找回 T-1/T-2

区分的唯一可靠信号：**next_page 是否存在 + 首页是否被塞满**。

---

## 第二步：翻页查找确定 `stop`

### 正常模式（日期可提取）

三层退出条件（哪个先触发就用哪个）：

| 退出条件 | 触发信号 | `stop` 结果 | 适用范围 |
|---------|---------|------------|---------|
| 窗口覆盖 | 最旧条目日期 ≤ T-2 | 当前翻页次数 | 所有分页类型 |
| 无新增 | `cur_count == prev_count` | 上一页（page-1） | 仅 append/加载更多 |
| 硬上限 | `page ≥ max_pages`（=10） | `max_pages`（有 T-1 时）或 1（无 T-1 时） | 所有分页类型 |

**判定流程**：

1. 翻页。提取本轮新增条目日期。
2. 找到 T-2 → 窗口覆盖退出，`stop = 当前页`。
3. append 模式下条目数无增长 → 无新增退出，`stop` 回退一页。
4. 翻到 `max_pages` → 硬上限退出。

**回退规则**：

| 翻页结果 | 处理 |
|---------|------|
| 无 T-1 且 无 T-2 | T-only 站点，回退模板 3.3 |
| 有 T-1 无 T-2 | 部分窗口覆盖，`stop = 当前页`，模板 3.4 |
| 有 T-1 且 有 T-2 | 完整窗口覆盖，`stop = 当前页`，模板 3.4 |

**翻页类型差异**：

| 翻页类型 | 行为 | 退出检测 |
|---------|------|---------|
| 传统翻页（页码跳转） | 新页替换旧页 | 只检查新页日期；不适用"无新增退出" |
| 加载更多（append） | 追加 N 条，列表单向增长 | 累积列表日期 + `cur_count == prev_count` |

---

### 降级模式（`date_unavailable = true`）

日期窗口不可用，退出仅依赖两个纯结构信号：

| 退出条件 | 触发信号 | 适用范围 |
|---------|---------|---------|
| 无新增（主） | `cur_count == prev_count` | 仅 append 模式 |
| 硬上限（兜底） | `page ≥ max_pages`（=5） | 所有模式 |

`max_pages` 从正常模式的 10 收紧为 5——因为没有日期验证窗口覆盖，更保守。

**`stop` 值由实测决定**：

| 实际情况 | `stop` |
|---------|--------|
| 点 1 次没新数据 | 1（回退 3.3） |
| 点 2 次到底 | 2 |
| 点 3 次到底 | 3 |
| 点 5 次还有新数据 | 5（触硬上限） |

---

## 关键约束

- **翻页次数必须有限**。不能以"按钮消失"为退出条件（append 按钮可能永不消失）。
- **T-1/T-2 不保证存在**。翻到 `max_pages` 仍找不到即接受现实，回退 3.3。
- **第 2 页即无新增** → 首页已是全部数据，回退 3.3。

---

## 判定示例速查

| 首页日期分布 | next_page | 首页填充度 | 判定 |
|-------------|-----------|-----------|------|
| 最旧 ≤ T-2 | — | — | 状态 A，3.3 |
| 只有 T | null | 任意 | C-Ⅰ，3.3 |
| 只有 T | XPath/scroll | 稀疏 | C-Ⅰ，3.3 |
| 只有 T | XPath/scroll | 满 | C-Ⅱ→B，进入翻页 |
| 有 T-1，最旧=T-1 | XPath | — | B，进入翻页 |
| 翻到第 N 页找到 T-2 | — | — | stop=N，3.4 |
| 翻到第 2 页无新增且无 T-2 | 加载更多 | — | 回退 3.3 |
| 翻到 10 页无 T-1 无 T-2 | — | — | T-only，回退 3.3 |
| 翻到 10 页有 T-1 无 T-2 | — | — | stop=10，3.4 |
| 日期不可提取 | XPath/scroll | 满 | 降级模式 |
| 日期不可提取 | null | 任意 | 不分页，3.3 |
